# PRD — Kali Tools GUI

**מסמך דרישות מוצר (Product Requirements Document)**
מחבר: הראלי דודאי · hareli26@gmail.com
גרסה: 1.0

---

## 1. חזון ומטרה
מערכת גרפית (Web) אחת שמאחדת את כלי ה‑Kali Linux הרצים תחת WSL2 ל‑Windows, ומאפשרת:
- **למשתמש מתחיל** — להריץ כלי פנטסט דרך טפסים ידידותיים, בלי לזכור דגלים ותחביר.
- **למשתמש מנוסה** — לתאר כוונה בשפה חופשית ולקבל תוכנית בדיקה מלאה שמתבצעת, מאומתת ומדווחת אוטומטית ע"י שכבת סוכנים.

## 2. קהל יעד
- בודקי חדירות, אנליסטים, סטודנטים לאבטחת מידע, חובבי CTF.
- משתמשים שעובדים על Windows עם Kali תחת WSL ומעדיפים ממשק גרפי על פני CLI.

## 3. בעיות שהמוצר פותר
| בעיה | פתרון |
|------|--------|
| קשה לזכור דגלים/תחביר של עשרות כלים | טופס דינמי לכל כלי עם תצוגת פקודה חיה |
| לא ברור באיזה כלי להשתמש לכל מטרה | סוכן Planner שממפה כוונה → כלים |
| הרצה, מעקב וסיכום ידניים ומפוזרים | סוכני Executor/Verifier/Reporter עם דוח אחוד |
| כלים דורשים root וסביבת WSL בעייתית | הרצה כ‑root ללא סיסמה + זיהוי כלים אוטומטי |

## 4. תכונות ליבה (Features)
### 4.1 מצב "כלים"
- דף בחירת כלי: 70 כלים ב‑11 קטגוריות, חיפוש וסינון, זיהוי מותקן/לא‑מותקן.
- מסך טופס: שדות דינמיים (טקסט/מספר/בחירה/צ'קבוקס), תצוגת פקודה חיה, שדה ארגומנטים מתקדם.
- מסך תוצאות נפרד: פלט חי (streaming), סטטוס, עצירה, העתקה, הורדה, הרצה חוזרת.
- התקנת כלי חסר בלחיצה (apt).

### 4.2 מצב "עוזר AI" (שכבת סוכנים)
- **Planner** — כוונה בשפה חופשית + מטרה → תוכנית שלבים (בחירת כלים, פקודות מוכנות, הצעות).
- **Executor** — הרצת השלבים ברצף.
- **Verifier** — ניתוח פלט + קוד יציאה → verdict (תקין/ממצאים/אזהרה/נכשל) + חילוץ ממצאים.
- **Reporter** — הפקת דוח Markdown מלא (סיכום מנהלים + פירוט + ממצאים).
- Playbooks: DNS, תת‑דומיינים, סריקת פורטים, בדיקת חדירות, SMB/Windows, Active Directory, API, מיפוי רשת, SSL/TLS, SQLi, WordPress, OSINT.

## 5. דרישות לא‑פונקציונליות
- **אפס תלויות** בצד השרת (ספריית פייתון סטנדרטית בלבד) — עובד על Kali מינימלי.
- **אבטחה** — פקודות נבנות בצד השרת כמערך `argv` ללא shell (הגנה מהזרקה).
- **RTL + עברית** מלא, ערכת נושא כהה.
- **מקומי בלבד** — השרת מאזין ל‑localhost; אין תעבורה החוצה מלבד הכלים עצמם.

## 6. ארכיטקטורה (תמצית)
```
דפדפן (Windows)  ──HTTP──►  server.py (Python, root, בתוך Kali WSL)
                                     ├── tools.json  (קטלוג כלים)
                                     ├── agents.py   (Planner/Verifier/Reporter)
                                     └── subprocess  ──►  כלי Kali אמיתיים
```

## 7. מדדי הצלחה (KPIs)
- זמן ממחשבה ("אני רוצה לבדוק X") עד הרצה: < 30 שניות.
- אחוז הכלים שמזוהים ומורצים ללא עריכת CLI ידנית: 100% מהקטלוג.
- דוח בדיקה מלא מופק אוטומטית בסיום כל משימה.

## 8. מחוץ לגבולות (Out of Scope) — גרסה נוכחית
- אימות מרובה‑משתמשים / הרשאות תפקידים.
- שמירת היסטוריית משימות במסד נתונים (כרגע בזיכרון בלבד).
- כלים אלחוטיים חיים (WSL ללא גישת monitor mode לחומרה).

## 9. סיכונים ומגבלות
- כלים אלחוטיים חיים לא יפעלו תחת WSL (רק aircrack-ng offline).
- דורש הרשאה חוקית למטרות — המוצר אינו אחראי לשימוש לרעה.

## 10. מפת דרכים עתידית
- אינטגרציית LLM מקומי (Ollama) לתכנון וניסוח דוחות חכם יותר.
- ייצוא דוח ל‑PDF/HTML ישירות מהממשק.
- שמירת פרויקטים/מטרות והשוואת סריקות לאורך זמן.
