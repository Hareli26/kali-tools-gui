## 3. מצב כלים
### שלב א — בחירת כלי
- חפש בתיבת החיפוש או סנן לפי קטגוריה (Recon, DNS, Network, Web, SMB, Passwords, SSL, Exploitation, Forensics, Wireless, Utilities).
- כרטיס ירוק = מותקן; כרטיס אפור = לא מותקן (לחיצה תציע התקנה).

### שלב ב — טופס והרצה
- מלא את השדות הרלוונטיים. **הפקודה המלאה מתעדכנת בזמן אמת** בצד.
- שדות חובה מסומנים ב‑`*`. שדה "ארגומנטים נוספים" מאפשר דגלים מתקדמים.
- לחץ **▶ הרץ**.

### שלב ג — תוצאות
- מסך נפרד עם פלט חי, סטטוס (רץ/הושלם/שגיאה), וכפתורי **עצור / העתק / הורד / הרץ שוב**.

## 4. מצב עוזר AI
ארבעה סוכנים עובדים ברצף מאחורי הקלעים:

```
כוונה חופשית ─► Planner ─► [אישור] ─► Executor ─► Verifier ─► Reporter ─► דוח
```

### שלב א — פרומט
- כתוב מה תרצה לבדוק בשפה חופשית, לדוגמה:
  *"בדיקת חדירות מקיפה לאתר"*, *"בדיקת DNS"*, *"אנומרציית SMB"*, *"מיפוי רשת מקומית"*.
- הזן מטרה: דומיין / IP / טווח CIDR / URL.
- לחץ **🧠 צור תוכנית**.

### שלב ב — תוכנית (Planner)
- הסוכן מציג את השלבים שבחר: שם הכלי, סיבה, הפקודה המדויקת והצעות בדיקה.
- **סמן/בטל** שלבים לפי הצורך. שלב שדורש root מסומן בתגית.
- לחץ **▶ הפעל משימה**.

### שלב ג — ביצוע ואימות (Executor + Verifier)
- כל שלב רץ ברצף; ליד כל שלב מופיע verdict: ✅ תקין · 🔎 ממצאים · ⚠️ אזהרה · ❌ נכשל.
- לחיצה על שלב פותחת את הפלט המלא שלו.

### שלב ד — דוח (Reporter)
- בסיום מופק **דוח Markdown** עם סיכום מנהלים, ממצאים עיקריים ופירוט שלבים.
- ניתן להעתיק או להוריד את הדוח.

## 5. דוגמאות מלאות
### דוגמה 1 — בדיקת DNS
מצב AI → כוונה: `בדיקת DNS` → מטרה: `example.com` → צור תוכנית → הפעל.
מתקבלים שלבי dig (A/MX/NS/TXT) + dnsenum + dnsrecon, ובסוף דוח.

### דוגמה 2 — בדיקת חדירות לאתר
כוונה: `בדיקת חדירות מקיפה לאתר` → מטרה: `https://target.example`.
תוכנית: nmap → nmap ‑sC → whatweb → nikto → nuclei → דוח מסכם.

### דוגמה 3 — כלי בודד ידני
מצב כלים → חפש `nmap` → מלא פורטים ‑sV ‑T4 → הרץ → צפה בתוצאות.
